# 🟦 1) Install Elasticsearch with Docker Compose  
This is the cleanest way to run Elasticsearch (and Kibana if you want) on your machine.

### **docker‑compose.yml**
```yaml
version: "3.8"

services:
  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:8.8.2
    container_name: es01
    environment:
      - discovery.type=single-node
      - xpack.security.enabled=false
    ports:
      - "9200:9200"
      - "9300:9300"
    volumes:
      - esdata:/usr/share/elasticsearch/data

volumes:
  esdata:
```

### **Start the service**
```bash
docker compose up -d
```

### **Verify**
```bash
curl http://localhost:9200
```

You should see cluster info in JSON.

# 🟩 2) Secure Elasticsearch with Passwords & TLS  
Elasticsearch 8.x includes **security enabled by default** (TLS + passwords).  
If you want to explicitly configure it:

### **1. Start Elasticsearch once to generate passwords**
```bash
docker run --rm \
  -e "discovery.type=single-node" \
  docker.elastic.co/elasticsearch/elasticsearch:8.8.2
```

The container prints:  
- `elastic` user password  
- CA certificate  
- HTTP certificate paths  

Copy these values.

### **2. Use the generated password**
Example:
```bash
curl -u elastic:YOUR_PASSWORD https://localhost:9200 --insecure
```

### **3. Using TLS certificates**
Elasticsearch generates:  
- `http_ca.crt` → for HTTPS  
- Node certificates → for cluster communication  

To use them with Docker Compose:

```yaml
environment:
  - xpack.security.enabled=true
  - xpack.security.http.ssl.enabled=true
  - xpack.security.http.ssl.key=certs/es01/es01.key
  - xpack.security.http.ssl.certificate=certs/es01/es01.crt
  - xpack.security.http.ssl.certificate_authorities=certs/ca/ca.crt
```

This is the same structure Elastic uses in their official examples.

# 🟧 3) Connect Applications (Python / Node.js / Java) to Elasticsearch  

Below are minimal working examples for each language.

---

## 🐍 Python (elasticsearch‑py)
```bash
pip install elasticsearch
```

```python
from elasticsearch import Elasticsearch

es = Elasticsearch("http://localhost:9200")

doc = {"title": "Hello", "content": "World"}
es.index(index="test", document=doc)

print(es.search(index="test", query={"match": {"content": "World"}}))
```

---

## 🟦 Node.js (official client)
```bash
npm install @elastic/elasticsearch
```

```js
const { Client } = require('@elastic/elasticsearch')

const client = new Client({ node: 'http://localhost:9200' })

async function run() {
  await client.index({
    index: 'test',
    document: { title: 'Hello', content: 'World' }
  })

  const result = await client.search({
    index: 'test',
    query: { match: { content: 'World' } }
  })

  console.log(result.hits.hits)
}

run()
```

---

## ☕ Java (High‑Level REST Client)
```xml
<dependency>
  <groupId>org.elasticsearch.client</groupId>
  <artifactId>elasticsearch-rest-high-level-client</artifactId>
  <version>7.17.0</version>
</dependency>
```

```java
RestHighLevelClient client = new RestHighLevelClient(
    RestClient.builder(new HttpHost("localhost", 9200, "http"))
);

IndexRequest request = new IndexRequest("test")
        .source(Map.of("title", "Hello", "content", "World"));

client.index(request, RequestOptions.DEFAULT);
```
