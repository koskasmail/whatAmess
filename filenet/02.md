## üõ†Ô∏è FileNet Command-Line Tools Cheat Sheet

This cheat sheet focuses on the most frequently used command-line utilities for administration and maintenance in a FileNet P8 environment, primarily utilizing the **Content Engine (CE)** and **Process Engine (PE)** tools.

| Tool/Command | Description | Common Use Case |
| :--- | :--- | :--- |
| **configmgr** (Configuration Manager) | GUI tool for configuring and deploying the **Content Engine**. | Initial setup, changing server/database connection settings, deploying the WAR file. |
| **wsimport** (WebSphere/Java Utility) | Used to generate Java artifacts (stubs/proxies) from the Content Engine WSDL file. | Generating code to communicate with the CE web service in a custom Java application. |
| **PE Configuration Manager** (Not CLI, but part of PE install) | Configures the **Process Engine** database, isolated regions, and connection points. | Setting up the workflow environment and connection to Content Engine. |
| **vwtool** | Command-line utility for managing and troubleshooting the **Process Engine** (workflows). | Checking region status, examining work items, killing misrouted workflows, managing configuration. |
| **vwlog** | Command-line utility for viewing **Process Engine** event logs. | Real-time monitoring of workflow events, troubleshooting workflow failures. |
| **vwtransfer** | Command-line utility to transfer workflow definitions between isolated regions or environments. | Deploying a new or updated workflow from a test to a production environment. |
| **sqlplus / db2 utility** | Native database command-line tools (used with caution\!). | Running pre-flight checks, backup/restore, checking connection details. (Direct modification is generally discouraged). |
| **Content Engine Startup Logs** | Located in the application server logs (e.g., `SystemOut.log` for WebSphere). | Checking for errors during CE server startup, identifying connectivity issues. |

This cheat sheet illustrates common operations using the **Content Engine Java API** (CE API) concepts, which are similar across other API implementations (like .NET or REST). The examples use conceptual code to show the **objects** and **methods** involved for typical actions.

| Action | API Concept/Objects Involved | Conceptual Code Example (Java API Style) |
| :--- | :--- | :--- |
| **Read (Fetch)** | `ObjectStore`, `Factory.Document`, `Document.fetchProperties()` | `Document doc = Factory.Document.fetchInstance(os, new Id("{...}"), null);`<br>`String title = doc.get_Name();` |
| **Create (Write)** | `Factory.Document`, `Document.set_Name()`, `Document.save()` | `Document doc = Factory.Document.createInstance(os, "Document");`<br>`doc.set_Name("My New Document");`<br>`doc.save(RefreshMode.REFRESH);` |
| **Add Content** | `ContentTransfer`, `Document.set_ContentElements()`, `Document.save()` | `ContentTransfer ct = Factory.ContentTransfer.createInstance();`<br>`ct.setCaptureSource(new FileInputStream("file.pdf"));`<br>`doc.set_ContentElements(Factory.ContentElementList.createList(ct));`<br>`doc.save(RefreshMode.REFRESH);` |
| **Change/Update** | `Document.fetchProperties()`, `Document.set_X()`, `Document.save()` | `Document doc = ... // fetch the document`<br>`doc.set_Name("Updated Name");`<br>`doc.set_FieldValue("CustomProp", "New Value");`<br>`doc.save(RefreshMode.REFRESH);` |
| **Check-Out** | `Document.checkOut()`, `Document.save()`, Returns a `Reservation` object. | `Reservation res = (Reservation) doc.checkOut(CheckOutType.EXCLUSIVE);`<br>`res.save(RefreshMode.REFRESH);` |

| **File in Folder** | `Folder`, `Folder.file()` | `Folder folder = ... // fetch the folder`<br>`folder.file(doc, AutoUniqueName.AUTO_UNIQUE, null);`<br>`folder.save(RefreshMode.REFRESH);` |
